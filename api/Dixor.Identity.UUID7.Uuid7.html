Remarks

Uuid7 serves as the central entry point for all 128-bit time-ordered identifier operations within the Dixor.Identity ecosystem.

Core capabilities exposed by this facade include:

  • Standard Generation: Zero-allocation creation of time-ordered primary keys (NewGuid()).
  • Monotonic Sequencing: Lock-free sequential ordering across same-millisecond bursts (NewMonotonicGuid()).
  • High-Efficiency Batching: Lazy iterator streaming for bulk database seeding and ETL pipelines (NewGuids(int)).
  • Inspection & Auditing: Zero-overhead extraction of embedded creation timestamps (GetTimestamp(Guid)).
  • Domain Formatting: Decorating identifiers with business prefixes and safe separators (NewString(string?, string?, string)).
  • Boundary Verification: Fast-path version validation and substring pattern parsing (IsValid(string), Parse(string)).

Unlike legacy NewGuid() (which emits random Version 4 values), identifiers produced by this class append sequentially at the end of database B-Tree indexes, eliminating page splits and fragmentation.

All generated identifiers strictly conform to RFC 9562.

NewGuid()

Generates a new RFC 9562 compliant UUID Version 7 identifier anchored to the current UTC timestamp.

public static Guid NewGuid()

Returns

Guid

A newly minted, chronologically sortable 128-bit primary key.

Examples

Guid primaryKey = Uuid7.NewGuid();

Remarks

The generated identifier combines a 48-bit millisecond UTC timestamp with 74 bits of cryptographically secure randomness. Hot execution paths utilize stack allocation to guarantee zero heap allocations.

NewGuids(int)

Generates a lazily evaluated sequence of multiple RFC 9562 UUID Version 7 identifiers.

public static IEnumerable<Guid> NewGuids(int count)

Parameters

count int

The number of identifiers to generate. Must be greater than zero.

Returns

IEnumerable<Guid>

An IEnumerable<T> yielding newly minted time-ordered identifiers upon enumeration.

Examples

// Stream 5,000 time-ordered keys directly into bulk processing
foreach (Guid primaryKey in Uuid7.NewGuids(5000))
{
    await dbContext.Orders.AddAsync(new Order { Id = primaryKey });
}

Remarks

Generation is deferred until the sequence is enumerated (e.g., via a foreach loop). This provides memory-efficient streaming for bulk inserts and database seeding without allocating large arrays in heap memory.

Exceptions

ArgumentOutOfRangeException

Thrown if count is less than or equal to zero.

NewMonotonicGuid()

Generates a new RFC 9562 UUID Version 7 identifier guaranteed to sort monotonically across same-millisecond bursts.

public static Guid NewMonotonicGuid()

Returns

Guid

A thread-safe, monotonically increasing 128-bit identifier.

Examples

Guid first = Uuid7.NewMonotonicGuid();
Guid second = Uuid7.NewMonotonicGuid();
// Guaranteed: first < second

Remarks

When NewGuid() is invoked thousands of times within the exact same millisecond, the resulting keys sort randomly due to lower-bit entropy. This method enforces mathematical sequence ordering by embedding a synchronized 16-bit sequence counter in the least significant bytes.
