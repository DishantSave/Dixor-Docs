Remarks

EmailValidationOptions allows callers to customize how EmailValidator validates email addresses.

The default instance follows commonly accepted internet standards and is suitable for most applications.

Unless explicitly changed, all properties use their documented default values.

AllowLocalDomains

Gets or sets a value indicating whether local-only domains (domains without a fully qualified domain name) are permitted.

public bool AllowLocalDomains { get; set; }

Property Value

bool

Remarks

Local domains are commonly used in internal networks, development environments and private infrastructures.

Examples of local domains: 

admin@localhost
user@internal
user@dev-machine

When disabled, domains that do not contain a dot separator are rejected unless they represent a valid fully qualified domain name.

Default value is false.
