Remarks

ValidationResult<TError> provides a consistent way for all validators in the Dixor.Validation library to communicate validation outcomes.

A successful validation returns Success(), while a failed validation contains an implementation-specific error value describing the reason for failure.

This type is immutable and allocation-free, making it suitable for high-performance validation scenarios.
