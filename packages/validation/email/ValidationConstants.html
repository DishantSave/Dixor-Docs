ValidationConstants Reference

The ValidationConstants class is an internal static utility that defines the architectural constraints and character literals used by the Dixor.Validation.Email library.

Overview

By centralizing these values, the library ensures that all internal validators (such as EmailParser, LocalPartValidator, and DomainValidator) use consistent boundaries, strictly adhering to RFC 5321 and RFC 5322 specifications.

Configuration Constants

Length Constraints

These limits define the structural boundaries of a valid email address as implemented by the library's parsing logic:

Constant Value Description
EmailMaxLength 254 Total length of the full email address.
EmailLocalPartMaxLength 64 Maximum length of the local-part.
EmailDomainMaxLength 253 Maximum length of the domain portion.
DomainLabelMaxLength 63 Maximum length of a single segment (label) in a domain.

Special Characters

Used by internal validators to avoid "magic strings" and ensure consistent character identification during tokenization and parsing:

  • Separators: At ('@'), `Dot` ('.').
  • Feature Flags: Plus ('+'), Quote ('"').
  • Domain Literals: OpenBracket ('['), CloseBracket (']').
