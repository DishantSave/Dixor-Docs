EmailValidationError Reference

The EmailValidationError enumeration defines a comprehensive set of reasons for email validation failures within the Dixor.Validation.Email library.

Overview

This enum is used primarily by ValidationResult<EmailValidationError> to provide granular, deterministic feedback when an email address fails validation checks.

Error Categories

Input & Basic Structure

  • None: Success.
  • Null, Empty, WhiteSpace: Issues with the raw input string.
  • TooLong: Total address exceeds length limits.
  • MissingAtSymbol, MultipleAtSymbols: Parsing failures for the address separator.

Local Part Issues

  • EmptyLocalPart, LocalPartTooLong: Size constraints.
  • LocalPartStartsWithDot, LocalPartEndsWithDot, ConsecutiveDots: Formatting rules for local parts.
  • InvalidLocalPartCharacter: Illegal symbols detected.

Domain & TLD Issues

  • EmptyDomain, DomainTooLong, DomainLabelTooLong: Size constraints.
  • DomainStartsWithDot, DomainEndsWithDot: Formatting rules.
  • DomainLabelStartsWithHyphen, DomainLabelEndsWithHyphen: Formatting rules for domain segments.
  • InvalidDomain, InvalidTopLevelDomain, InvalidDomainCharacter: Syntax and TLD validity checks.

Policy & Advanced Features

  • QuotedLocalPartNotAllowed, InvalidQuotedLocalPart: Handling of quoted syntax.
  • PlusAddressingNotAllowed: Handling of sub-addressing.
  • InvalidIpDomain: IP literal validation.
  • InternationalCharactersNotAllowed: Unicode/EAI configuration checks.
  • NotAllowed: General policy rejection.
