EmailParser Reference

The EmailParser is the central engine of the Dixor.Validation.Email library, coordinating the end-to-end processing pipeline for email validation and decomposition.

Overview

The parser transforms a raw string into a structured EmailInfo object by executing a series of validation stages. If any stage fails, it returns a specific EmailValidationError through the ValidationResult.

The Parsing Pipeline

Parsing occurs in the following sequential order:

  1. Input Validation: Ensures the string is not null, empty, or just whitespace.
  2. Normalization: Canonicalizes the email using the EmailNormalizer (trimming, Unicode normalization, case folding).
  3. Decomposition: Splits the string at the @ symbol, ensuring exactly one separator exists.
  4. Component Validation:
    • Invokes ILocalPartValidator to verify the local part syntax.
    • Invokes IDomainValidator to verify domain syntax.
  5. Metadata Extraction: Constructs the EmailInfo object, including:
    • Domain hierarchy (TLD, SLD, Subdomain).
    • Feature detection (Plus addressing, Quoted local parts, IP literals).
