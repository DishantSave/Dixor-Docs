EmailValidator Documentation

Overview

EmailValidator is the primary entry point for working with email addresses in the Dixor.Validation library. It provides a static, thread-safe API to validate, normalize, and inspect email address components.

API Reference

1. Email Validation Endpoints

  • Function 1: IsValid Determines whether the specified email address is valid.
  • Function 2: Validate Validates the specified email address and returns a detailed ValidationResult.

2. Email Normalizing Endpoints

  • Function 3: Normalize Normalizes the specified email address to its canonical form.
  • Function 4: TryNormalize Attempts to normalize the specified email address, returning the result via an out parameter.

3. Email Information Endpoints

  • Function 5: GetInfo Retrieves detailed information (as an EmailInfo instance) about the specified email address.

4. Email Component Endpoints

  • Function 6: GetLocalPart Retrieves the local part of the specified email address.
  • Function 7: TryGetLocalPart Attempts to retrieve the local part of the specified email address.
  • Function 8: GetDomain Retrieves the domain portion of the specified email address.
  • Function 9: TryGetDomain Attempts to retrieve the domain portion of the specified email address.
  • Function 10: GetSubDomain Retrieves the subdomain portion if one exists.
  • Function 11: TryGetSubDomain Attempts to retrieve the subdomain portion if one exists.

5. Email Domain Matching Endpoints

  • Function 12: HasDomain Checks if the email belongs to a specific domain.
  • Function 13: HasAnyDomain Checks if the email belongs to any domain in a provided collection.
  • Function 14: HasTopLevelDomain Checks if the email uses a specific top-level domain (TLD).

6. Email Comparison Endpoints

  • Function 15: AreEqual Determines if two email addresses are equivalent after normalization.
