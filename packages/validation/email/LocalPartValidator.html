LocalPartValidator Reference

The LocalPartValidator is an internal service within the Dixor.Validation.Email library responsible for syntactical and policy-based verification of the "local part" of an email address (the segment before the @ symbol).

Overview

This validator ensures the local part adheres to formatting rules defined by RFC 5321/5322 and respects the custom EmailValidationOptions provided by the consumer.

Validation Rules

  1. Length: Must not exceed EmailLocalPartMaxLength.
  2. Dot Positioning:
    • No consecutive dots (..).
    • Cannot start or end with a dot.
  3. Policy Controls:
    • Quoted Strings: Checks if the local part is enclosed in ""; rejected if AllowQuotedLocalPart is false.
    • Plus Addressing: Checks for + character; rejected if AllowPlusAddressing is false.
  4. Character Set:
    • Strictly validates against a whitelist of ASCII special characters allowed in email local parts.
    • If AllowInternational is enabled, it permits Unicode characters (non-ASCII) beyond the standard set.
