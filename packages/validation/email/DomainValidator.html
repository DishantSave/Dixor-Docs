DomainValidator Reference

The DomainValidator is an internal service within the Dixor.Validation.Email library responsible for the syntactical verification of the domain portion of an email address.

Overview

This validator ensures that the domain part of an email adheres to structural and length requirements defined by internet standards. It functions independently of the local part validation.

Validation Logic

The validator executes three primary checks:

  1. Length Validation: Ensures the total domain string length does not exceed EmailDomainMaxLength.
  2. Structural Validation (ValidateLabels):
    • Prevents .. sequences.
    • Prevents domain names starting or ending with a dot.
    • Ensures every individual label (part separated by dots) is non-empty and follows length constraints.
    • Validates label boundary characters (prevents starting or ending with a hyphen).
  3. Character Validation:
    • Permits ASCII letters, digits, dots, and hyphens by default.
    • If AllowInternational is set to true in EmailValidationOptions, it permits Unicode characters (non-ASCII).
