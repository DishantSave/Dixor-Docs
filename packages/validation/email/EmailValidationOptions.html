EmailValidationOptions Reference

The EmailValidationOptions class provides a configuration object that allows developers to customize the validation logic of the EmailValidator class.

Overview

By modifying the properties of an EmailValidationOptions instance, you can tailor the validation process to meet specific business requirements, such as restricting disposable emails or enabling specific formats like plus addressing.

Configuration Properties

Property Type Default Description
AllowInternational bool true Permits international (Unicode/EAI) email addresses.
AllowPlusAddressing bool true Permits plus addressing (e.g., user+tag@domain.com).
AllowQuotedLocalPart bool false Permits quoted strings in the local part (e.g., "john smith"@domain.com).
AllowLocalhost bool false Permits localhost as a domain.
AllowIpDomain bool false Permits IP address domain literals (e.g., user@[127.0.0.1]).
AllowDisposableDomains bool true Permits domains associated with temporary/disposable email services.
AllowPublicDomains bool true Permits public email providers (e.g., Gmail, Outlook).
AllowRoleBasedAddresses bool true Permits role-based accounts (e.g., admin@, support@).
TrimInput bool true Automatically trims leading/trailing whitespace before validation.
IgnoreDomainCase bool true Treats domain names as case-insensitive.

Usage Example

var options = new EmailValidationOptions
{
    AllowDisposableDomains = false,
    AllowPlusAddressing = false
};

bool isValid = EmailValidator.IsValid("test@mailinator.com", options); // Returns false
