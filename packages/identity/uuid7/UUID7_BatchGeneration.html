High-Efficiency Batch UUIDv7 Generation

📚 Navigation: Home > Dixor.Identity > Batch Generation

Dixor.Identity provides high-performance, deferred generation of multiple UUIDv7 identifiers through the Uuid7.NewGuids(int count) facade.

IEnumerable<Guid> batchIds = Uuid7.NewGuids(count: 10000);

Architectural Performance & Lazy Evaluation

The underlying Uuid7BatchGenerator utilizes C# iterator state machines (yield return) to stream items lazily upon enumeration. This maintains \(O(1)\) initial memory overhead regardless of the requested batch size, eliminating heap exhaustion and Garbage Collector spikes during ETL migrations or mass data seeding.

// Internally delegates generation per item upon enumeration
public IEnumerable<Guid> Generate(int count)
{
    ArgumentOutOfRangeException.ThrowIfLessThanOrEqual(count, 0);

    for (int i = 0; i < count; i++)
    {
        yield return _generator.Generate();
    }
}

API Usage & Examples

Basic Streaming

using Dixor.Identity.UUID7;

// Stream time-ordered keys directly into a database bulk inserter
foreach (Guid primaryKey in Uuid7.NewGuids(5000))
{
    await dbContext.Orders.AddAsync(new Order { Id = primaryKey });
}

Materialization

Materialize explicitly only if your downstream code or serialization layer demands an in-memory collection:

List<Guid> orderIds = Uuid7.NewGuids(count: 500).ToList();
Guid[] idArray      = Uuid7.NewGuids(count: 100).ToArray();
