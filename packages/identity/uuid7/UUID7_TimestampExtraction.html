Timestamp Extraction (Uuid7.GetTimestamp)

📚 Navigation: Home > Dixor.Identity > Timestamp Extraction

One of the most powerful architectural advantages of RFC 9562 UUID Version 7 over traditional random GUIDs is the embedded 48-bit millisecond Unix timestamp.

Instead of forcing your database schemas to store explicit, redundant creation date columns (CreatedAt DATETIME2), Dixor.Identity lets you extract exact creation timestamps directly from the identifier itself with zero memory allocation:

Dixor.Identity enables zero-allocation extraction of embedded 48-bit millisecond Unix timestamps directly from RFC 9562 UUID Version 7 identifiers via Uuid7.GetTimestamp and Uuid7.TryGetTimestamp.

using Dixor.Identity.UUID7;

Guid orderId = Uuid7.NewGuid();
DateTimeOffset createdAt = Uuid7.GetTimestamp(orderId);

Why Extract Timestamps?

  1. Schema & Storage Optimization: Eliminate secondary index storage overhead on dedicated timestamp columns by deriving entity creation chronology straight from primary keys.
  2. Event Sourcing Analysis: Instantly order decoupled domain events across distributed message brokers (Kafka, RabbitMQ) without relying on synchronized network clocks.
  3. Audit & Telemetry Recovery: Inspect logs, correlation IDs, or customer support tokens to identify exact request inception times during debugging.

Bitwise Decoding Anatomy

When Uuid7.GetTimestamp() executes, it reads the first 6 bytes (48 bits) of the 16-byte UUID structure in Network Byte Order (Big-Endian):

Byte 0      Byte 1      Byte 2      Byte 3      Byte 4      Byte 5
[Bits 47..40][Bits 39..32][Bits 31..24][Bits 23..16][Bits 15..8] [Bits 7..0]
└────────────────────────┬─────────────────────────────────────────┘
                         ▼
        48-Bit Integer (Milliseconds since Epoch)

Direct bitwise left-shifting reconstructs the millisecond Unix timestamp (ToUnixTimeMilliseconds), which is immediately parsed into a standard UTC DateTimeOffset.

API Overloads & Boundary Safety

Method Throws on Failure? Best Used For
Uuid7.GetTimestamp(Guid) ✅ Yes (ArgumentException) Trusted domain entities, database records.
Uuid7.TryGetTimestamp(Guid, out DateTimeOffset) ❌ No (Returns bool) Untrusted REST payloads, query strings, webhooks. 
// Safe extraction pattern for incoming HTTP requests
if (Uuid7.TryGetTimestamp(request.CorrelationId, out DateTimeOffset requestTime))
{
    Console.WriteLine($"Request initiated at: {requestTime:O}");
}
