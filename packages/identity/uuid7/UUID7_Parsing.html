UUID Parsing & Text Extraction (Uuid7.Parse & Uuid7.TryParse)

📚 Navigation: Home > Dixor.Identity > Parsing

In enterprise telemetry, application logs, and external messaging pipelines, identifiers rarely arrive as clean, isolated 36-character strings. They are frequently wrapped inside log headers, domain prefixes, or JSON payloads:

[INFO] Order processed successfully. CorrelationId=ORD-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10

While the standard .NET Guid.Parse() fails immediately on strings containing surrounding characters, Dixor.Identity incorporates a compiled regex extraction engine capable of locating and parsing UUIDs embedded inside arbitrary text:

using Dixor.Identity.UUID7;

string logLine = "[INFO] CorrelationId=ORD-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10-EU";

// Automatically extracts the embedded canonical UUID pattern
Guid correlationId = Uuid7.Parse(logLine);
Console.WriteLine(correlationId); // Output: 0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10

How Embedded Extraction Works

When Uuid7.Parse or Uuid7.TryParse executes, it evaluates incoming text against a pre-compiled regular expression matching standard 8-4-4-4-12 hexadecimal framing:

xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx


Search Pipeline:
Input Text: "Customer_0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10_US"
                  │
                  ▼ (Regex Match Engine)
Locates:    "0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10"
                  │
                  ▼ (Guid.TryParse)
Returns:     Guid Struct
  • Compiled Performance: The internal engine compiles the pattern strictly once (RegexOptions.Compiled), guaranteeing high-speed execution across millions of log rows or stream events.
  • First Match Wins: If an input string contains multiple embedded UUIDs, the parser extracts strictly the first occurrence.

API Overloads & Exception Handling

Method Throws on Failure? Returns Best Used For
Uuid7.Parse(string) ✅ Yes (FormatException) Guid Trusted internal strings, configuration values, guaranteed schemas.
Uuid7.TryParse(string, out Guid) ❌ No bool Untrusted user input, external webhooks, parsing messy log files.
Warning

Null Protection: Both overloads enforce early null checking (ArgumentNullException.ThrowIfNull). Passing a null reference will throw immediately rather than failing silently.

High-Throughput Log Processing Example

When ingesting unstructured application logs or external CSV files, use TryParse inside loop iterations to prevent exception overhead:

foreach (string rawLogRow in batchLogRows)
{
    // Skip unparseable log rows silently with zero exception cost
    if (Uuid7.TryParse(rawLogRow, out Guid extractedId))
    {
        await telemetryIndexer.IndexRecordAsync(extractedId);
    }
}
