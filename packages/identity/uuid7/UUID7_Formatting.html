Business Identifier Formatting (Uuid7.NewString)

📚 Navigation: Home > Dixor.Identity > Formatting

While a raw 128-bit UUID Version 7 string (0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10) guarantees uniqueness and time-ordering, it lacks visual context in enterprise logs, customer invoices, and microservice APIs.

Dixor.Identity provides built-in, allocation-conscious string decoration to wrap raw UUIDs in clean domain context without sacrificing uniqueness:

string orderReference = Uuid7.NewString(prefix: "ORD", suffix: "US");
// Output: ORD-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10-US

Common Use Cases

Transforming raw GUIDs into readable business references significantly simplifies system telemetry, support ticketing, and multi-tenant routing:

Domain Example API Call Generated Output
Order Management Uuid7.NewString("ORD") ORD-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10
Invoice Tracking Uuid7.NewString("INV", "EU") INV-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10-EU
Multi-Tenancy Uuid7.NewString("TENANT1", separator: "_") TENANT1_0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10

Guardrails & Validation Rules

To prevent database truncation errors, malformed URLs, and unreadable strings, the internal Uuid7Formatter enforces strict boundary rules:

  1. Length Limits: Prefixes and suffixes cannot exceed 50 characters each (MaxPrefixLength / MaxSuffixLength). Passing larger strings throws an ArgumentOutOfRangeException.
  2. Safe Separators: To protect file systems, URLs, and shell scripts from unescaped wildcards or control characters, the separator must consist of exactly one character from our safe allowlist:
-   _   :   ~   |   /   \   {   }   [   ]   <   >
  1. Null/Whitespace Tolerance: Passing null, "", or " " as a prefix/suffix is gracefully ignored without appending hanging separators.

Compile-Time Safety: Roslyn Analyzers

When formatting inputs are known at compile time, Dixor.Identity validates your parameters inside the Visual Studio/Rider IDE before you even compile the project:

Diagnostic Rule Description Trigger Condition
DIXOR002 Prefix exceeds limit Passing a constant string > 50 characters as prefix.
DIXOR003 Suffix exceeds limit Passing a constant string > 50 characters as suffix.
DIXOR004 Invalid separator Passing an unsupported character (e.g., *, ?) or multi-char string. 
// ❌ COMPILE ERROR (DIXOR004): Separator '*' is not allowed
var badId = Uuid7.NewString("ORD", separator: "*");

// ✅ COMPILE SUCCESS: Valid safe separator
var goodId = Uuid7.NewString("ORD", separator: "_");
