Business Identifier Formatting (Uuid7.NewString)

📚 Navigation: Home > Dixor.Identity > Formatting

Dixor.Identity provides allocation-conscious string decoration to wrap raw UUIDs in clean domain context via the Uuid7.NewString facade without sacrificing uniqueness.

string orderReference = Uuid7.NewString(prefix: "ORD", suffix: "US");
// Output: ORD-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10-US

Common Use Cases

Transforming raw GUIDs into readable business references significantly simplifies system telemetry, support ticketing, and multi-tenant routing:

Domain Example API Call Generated Output
Order Management Uuid7.NewString("ORD") ORD-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10
Invoice Tracking Uuid7.NewString("INV", "EU") INV-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10-EU
Multi-Tenancy Uuid7.NewString("TENANT1", separator: "_") TENANT1_0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10

Formatting Guardrails & Validation

The underlying Uuid7Formatter enforces strict boundary rules to protect URLs, file paths, and database columns:

  1. Length Limits: Prefixes and suffixes cannot exceed 50 characters (MaxPrefixLength / MaxSuffixLength), throwing an ArgumentOutOfRangeException if exceeded.
  2. Safe Separators: Separators must consist of exactly one character from a strict allowlist:
-   _   :   ~   |   /   \   {   }   [   ]   <   >
  1. Null/Whitespace Tolerance: Passing null, "", or whitespace as a prefix or suffix is gracefully ignored without adding hanging delimiters.
