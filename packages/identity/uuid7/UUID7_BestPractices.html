UUIDv7 Production Best Practices

📚 Navigation: Home > Dixor.Identity > Best Practices

Adopting UUID Version 7 optimizes database indexing and chronological tracking. This guide outlines essential best practices for database storage, API boundary safety, and high-concurrency generation.

1. Database Storage & Schema Design

Always store UUIDs as native 16-byte binary types rather than text strings (VARCHAR(36)), which double disk storage consumption and fragment database B-Tree index pages.

Database Engine Recommended Column Type Why?
SQL Server UNIQUEIDENTIFIER Optimized for sequential leaf appends.
PostgreSQL UUID First-class binary handling.
MySQL / MariaDB BINARY(16) Fixed-length raw binary layout.

2. Choosing the Generation Strategy

  • Standard Workloads (Uuid7.NewGuid()): Zero-allocation time-ordered IDs for REST APIs and entity creation.
  • High-Throughput Concurrency (Uuid7.NewMonotonicGuid()): Strict mathematical sorting across same-millisecond bursts for event sourcing and CQRS.
  • Bulk Operations (Uuid7.NewGuids()): Memory-efficient lazy streaming for data seeding and ETL migrations.

3. API Boundary Safety & Input Validation

Avoid exception-driven parsing on untrusted user input. Use safe validation patterns at API controllers:

// ❌ BAD: Exception-driven flow incurs severe CPU overhead on bad requests
try 
{
    Guid id = Uuid7.Parse(rawInput);
} 
catch (FormatException) { /* Handle error */ }

// ✅ GOOD: Fast, allocation-free validation pattern
if (!Uuid7.TryParse(rawInput, out Guid validId) || !Uuid7.IsValid(validId))
{
    return Results.BadRequest("Invalid RFC 9562 UUIDv7 identifier provided.");
}

4. Architectural Patterns & Domain Formatting

  • Decouple Generation: Wrap static generation calls behind an IIdGenerator interface to keep unit tests deterministic.
  • Use Built-In Formatting: Use Uuid7.NewString(prefix, suffix) to wrap identifiers in domain context cleanly without manual string allocation overhead.

5. Security & Temporal Considerations

  • Not for Authorization: Embedded timestamps mean observers can deduce creation times; never use UUIDs as secret tokens.
  • Preserve UTC: Extracted timestamps (Uuid7.GetTimestamp) return UTC DateTimeOffset. Keep UTC across services and convert locally only at the UI layer.

Anti-Pattern Production Best Practice

Looping Uuid7.NewGuid() for mass inserts Use Uuid7.NewGuids(count) for lazy streaming.
Using Parse() on HTTP inputs Use TryParse() and IsValid().
