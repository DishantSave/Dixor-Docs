Dixor.Identity

📚 Navigation: Home > Dixor.Identity Overview

Dixor.Identity delivers modern, allocation-conscious identifier generation for .NET.

While standard .NET applications heavily rely on Guid.NewGuid(), that API generates UUID Version 4 values. Because Version 4 identifiers are completely random, inserting them as primary keys into clustered database tables forces B-Tree index page splits, random disk I/O, cache misses, and severe database fragmentation under high-throughput workloads.

Dixor.Identity solves this problem by providing a complete, zero-allocation implementation of RFC 9562 UUID Version 7. By embedding a 48-bit millisecond Unix timestamp in the most significant bits alongside 74 bits of cryptographically secure randomness, Version 7 identifiers naturally sort in chronological order while guaranteeing global collision resistance.

Core Capabilities at a Glance

All operations are centralized cleanly under the static Uuid7 facade:

using Dixor.Identity.UUID7;

// 1. Standard Generation: Mint a zero-allocation, time-ordered primary key
Guid primaryKey = Uuid7.New();

// 2. Monotonic Sequencing: Guarantee strict mathematical order across same-millisecond bursts
Guid eventId = Uuid7.NewMonotonic();

// 3. Batch Streaming: Lazily yield 5,000 keys for ETL pipelines without heap bloat
IEnumerable<Guid> batch = Uuid7.NewGuids(count: 5000);

// 4. Timestamp Extraction: Decode inception chronology directly from the ID payload
DateTimeOffset createdAt = Uuid7.GetTimestamp(primaryKey);

// 5. Domain Formatting: Decorate keys with business prefixes and safe separators
string invoiceRef = Uuid7.NewString(prefix: "INV", suffix: "EU");
// Output: INV-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10-EU

// 6. Safe Parsing & Validation: Verify untrusted HTTP headers or log strings
if (Uuid7.IsValid("ORD-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10"))
{
    Guid orderId = Uuid7.Parse("ORD-0197d8ef-f4c6-7a35-99e3-1d4a7e3c8e10");
}
