Numeric Code Generation (NumericCode.Generate)

📚 Navigation: Home > Dixor.Identity > NumericCode

Dixor.Identity provides cryptographically secure numeric codes consisting exclusively of numeric characters (0–9) via the NumericCode.Generate() facade, suitable for verification codes, one-time passwords (OTPs), and authentication tokens.

using Dixor.Identity;

// Generate a secure numeric code of default length
string code = NumericCode.Generate();

What are Numeric Codes & Why Use Them?

Numeric Codes are secure, purely numerical character sequences restricted to digits 0–9.

Key Advantages

  • Optimized for Human Input: Digits-only codes are ideal for communication channels where letters can be easily misread or mistyped (such as SMS verification, voice prompts, and hardware tokens).
  • Predictable Formatting: Fixed-length numeric strings integrate seamlessly with standard multi-factor authentication (MFA) UIs and PIN entry boxes.
  • Cryptographic Security: Built using high-entropy random entropy providers (RandomProvider), Dixor.Identity ensures that generated tokens are unpredictable and secure against brute-force guessing attacks.

Common Use Cases

  • One-Time Passwords (OTPs): Temporary authentication tokens delivered via SMS, email, or authenticator apps.
  • Account Verification Pins: Short numeric codes used during user registration, phone confirmation, or password reset workflows.
  • Transaction Authorization Tokens: Secure PIN codes required to confirm financial transfers or sensitive user actions.

Algorithm & Execution Mechanics

The underlying generation engine delegates to shared string generation infrastructure using numerical character sets and optimized memory management:

  1. Character Set Mapping: Selects characters uniformly from the numeric alphabet (0–9).
  2. Optimized Buffer Allocation: For high-throughput performance, strings up to 256 characters are allocated directly on the execution stack (stackalloc char[length]), completely bypassing heap memory allocations and eliminating Garbage Collector pressure.
  3. Cryptographic Entropy: Random character selection utilizes shared secure entropy sources (RandomProvider), guaranteeing high entropy and unpredictability.
// Example custom length invocation with parameter validation
string customCode = NumericCode.Generate(length: 6);

Guardrails & Validation Rules

To protect system integrity and prevent resource exhaustion, the underlying NumericCodeService enforces strict bounds:

  • Length Constraints: Passing a length less than or equal to zero, or greater than Constants.MaxNumericCodeLength, triggers an ArgumentOutOfRangeException.
