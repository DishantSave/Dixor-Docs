Base62 Identifier Generation (Base62.Generate)

📚 Navigation: Home > Dixor.Identity > Base62

Dixor.Identity provides cryptographically secure, compact, and URL-safe Base62 identifiers consisting of numeric characters (0–9), uppercase alphabetic characters (A–Z), and lowercase alphabetic characters (a–z) via the Base62.Generate() facade.

using Dixor.Identity;

// Generate a secure Base62 identifier of default length
string identifier = Base62.Generate();

What is Base62 & Why Use It?

Base62 is a binary-to-text encoding format that uses 62 alphanumeric characters: numeric digits 0–9, uppercase letters A–Z, and lowercase letters a–z.

Key Advantages

  • Higher Encoding Density: By incorporating lowercase characters alongside numbers and uppercase letters, Base62 packs more information into fewer characters than Base36, resulting in shorter identifiers for the same payload size.
  • URL-Safe & Case-Sensitive: Base62 omits special symbols, punctuation marks, and whitespace, making it entirely safe for inclusion in URLs, URI query parameters, and REST routing paths. Because it is case-sensitive, it preserves high information capacity.
  • Cryptographic Security: Built on top of secure entropy providers (RandomProvider), Dixor.Identity ensures that Base62 identifiers are unpredictable and globally collision-resistant.

Common Use Cases

  • Database & Entity Slugs: Clean, concise resource handles for web application routes (e.g., /users/3GZ9k2X).
  • Short-Link Generators: High-density short codes for URL shortening services where brevity and link readability are critical.
  • Distributed Tracking IDs: Compact correlation or transaction tokens that pass seamlessly through strict API gateways and log parsers.

Algorithm & Execution Mechanics

The underlying generation engine relies on high-entropy cryptographic random providers and efficient buffer allocation:

  1. Character Set Mapping: Selects characters uniformly from the Base62 alphabet (0–9, A–Z, a–z), providing a higher encoding density than Base36.
  2. Optimized Buffer Allocation: For high-throughput performance, strings up to 256 characters are allocated directly on the execution stack (stackalloc char[length]), completely bypassing heap memory allocations and eliminating Garbage Collector pressure.
  3. Cryptographic Entropy: Random character selection utilizes shared secure entropy sources (RandomProvider), guaranteeing collision resistance and unpredictability.
// Example custom length invocation with parameter validation
string customId = Base62.Generate(length: 16);

Guardrails & Validation Rules

To protect system integrity and prevent resource exhaustion, the underlying Base62Service enforces strict bounds:

  • Length Constraints: Passing a length less than or equal to zero, or greater than Constants.MaxBase62Length, triggers an ArgumentOutOfRangeException.
