Fixed-Index Bit Mapping Strategy

To eliminate offset drift and ensure exact alignment with the formal ULID specification, components avoid dynamic accumulator loops during serialization and instead rely on Fixed-Index Bit Mapping.

Encoding Map (128-bit Binary → 26 Base32 Characters)

  • Characters 0–9: Map directly to the 48-bit timestamp and upper segment bits.
  • Characters 10–25: Map across the 80-bit cryptographic randomness block.
  • Padding: 130 bits total capacity handles 128 data bits plus 2 zero-padded trailing bits on the final character chunk.

Decoding Map (26 Base32 Characters → 16-Byte Array)

  • Validates length constraints (\(26\text{ chars}\)) and leading overflow bounds (first character index value \(\le 7\)).
  • Translates Crockford Base32 characters (including human error-tolerant aliases like I \(\rightarrow\) 1, L \(\rightarrow\) 1, and O \(\rightarrow\) 0) back into exact byte segments via inverse bit shifting.
