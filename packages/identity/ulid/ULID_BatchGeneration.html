ULID Batch Generation (Ulid.Generate(int count))

📚 Navigation: Home > Dixor.Identity > Batch Generation

Dixor.Identity provides high-efficiency batch generation for creating multiple Universally Unique Lexicographically Sortable Identifiers (ULIDs) in a single operation via the Ulid.Generate(int count) facade.

using Dixor.Identity.ULID;

// Generate a lazily evaluated sequence of 500 ULIDs for bulk processing
foreach (var id in Ulid.Generate(500))
{
    Console.WriteLine(id);
}

Architectural Mechanics & Lazy Evaluation

The batch generation mechanism is designed to handle bulk inserts, seed data generation, and high-throughput data streams efficiently:

  1. Deferred Execution: The batch generator returns an IEnumerable<string>, leveraging lazy evaluation so that identifiers are generated on-demand as the consumer iterates through the collection.
  2. Delegated Generation: The underlying batch processor (UlidBatchGenerator) iterates through the requested count and repeatedly invokes the single-value generation engine to ensure every identifier receives a fresh timestamp and cryptographically secure random component.

Input Validation & Error Handling

To safeguard against invalid allocations or infinite loop vectors, the batch generator enforces strict input boundary checks:

  • Argument Guards: Uses high-performance guard clauses (ArgumentOutOfRangeException.ThrowIfNegativeOrZero(count)) to immediately reject any batch requests where the count is zero or negative.
// Throws an ArgumentOutOfRangeException if count <= 0
var invalidBatch = Ulid.Generate(0);
