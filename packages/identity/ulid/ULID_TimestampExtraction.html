ULID Timestamp Extraction (Ulid.GetTimestamp / Ulid.TryGetTimestamp)

📚 Navigation: Home > Dixor.Identity > Timestamp Extraction

Dixor.Identity provides high-performance functionality for extracting embedded creation timestamps directly from Universally Unique Lexicographically Sortable Identifiers (ULIDs) without querying external metadata stores, via Ulid.GetTimestamp and Ulid.TryGetTimestamp.

using Dixor.Identity.ULID;

string id = "01J5J8Y9Q6A8K5J8X3Z5N9Y2KP";

// Extract timestamp directly (throws exception if format is invalid)
DateTime createdUtc = Ulid.GetTimestamp(id);

// Safely attempt timestamp extraction without throwing exceptions
if (Ulid.TryGetTimestamp(id, out DateTime timestamp))
{
    Console.WriteLine($"Created at: {timestamp} UTC");
}

Architectural Mechanics & Timestamp Decoding

A ULID encodes its temporal creation point within the first 48 bits of its 128-bit structure:

+-------------------+-------------------+
| 48-bit Timestamp  | 80-bit Randomness |
| (Bytes 0–5)       | (Bytes 6–15)      |
+-------------------+-------------------+
  1. Big-Endian Decoding: The extraction engine reads the first six bytes (ulidBytes[0] through ulidBytes[5]) as a 48-bit big-endian Unix timestamp stored in milliseconds.
  2. Byte Bit-Shifting: The bytes are combined using bitwise shifting (((long)ulidBytes[0] << 40) | ...) to reconstitute the total millisecond epoch value.
  3. UTC DateTime Conversion: The reconstructed millisecond value is converted into a UTC DateTime instance via DateTimeOffset.FromUnixTimeMilliseconds(timestamp).UtcDateTime.

Validation & Safety Checks

To maintain robustness during parsing and extraction:

  • Length & Byte Verification: The extractor ensures the input binary representation contains exactly 16 bytes (or verifies the string via parsing/validation layers), throwing an ArgumentException if length constraints are violated.
  • Safe Extraction Guards: TryGetTimestamp validates the string format beforehand, returning false gracefully if malformed input is supplied.
