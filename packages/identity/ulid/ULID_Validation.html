ULID Validation Rules (Ulid.IsValid)

📚 Navigation: Home > Dixor.Identity > Validation Rules

Dixor.Identity provides high-performance, syntax-level validation for verifying whether a supplied string representation conforms to the official Universally Unique Lexicographically Sortable Identifier (ULID) specification via Ulid.IsValid.

using Dixor.Identity.ULID;

// Validate a ULID string without throwing exceptions
bool isValid = Ulid.IsValid("01J5J8Y9Q6A8K5J8X3Z5N9Y2KP");

Architectural Mechanics & Performance

The validation engine is engineered to perform quick syntactic verification entirely on memory spans without triggering heap allocations or incurring the overhead of full cryptographic parsing:

  1. Span-Based Inspection: Accepts a ReadOnlySpan<char> to validate input strings directly.
  2. Optimized Lookup: Utilizes a high-performance HashSet<char> containing all valid Crockford Base32 characters for O(1) character membership checking.

Validation Rules Enforced

A string representation must successfully pass all of the following rules to be considered a valid ULID:

  1. Exact Length Constraint: The input value must contain exactly 26 characters.
  2. Alphabet Membership: Every character must belong to the Crockford Base32 alphabet (0-9, A-H, J-K, M-N, P-T, V-Z, case-insensitive). Visually ambiguous aliases (I, L, and O) are also supported.
  3. Leading Overflow Bounds: The first character's decoded numeric value must not exceed 7 to prevent bit-range overflows beyond the 128-bit boundary.
