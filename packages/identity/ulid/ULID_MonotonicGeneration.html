ULID Monotonic Generation (Ulid.GenerateMonotonic)

📚 Navigation: Home > Dixor.Identity > Monotonic Generation

Dixor.Identity provides a high-throughput, thread-safe implementation for generating monotonically increasing Universally Unique Lexicographically Sortable Identifiers (ULIDs) via the Ulid.GenerateMonotonic() facade.

using Dixor.Identity.ULID;

// Generate strictly increasing ULIDs during high-frequency same-millisecond bursts
string firstId = Ulid.GenerateMonotonic();
string secondId = Ulid.GenerateMonotonic();

The Same-Millisecond Ordering Challenge

Standard timestamp-based generation relies on the system clock. When high-throughput applications generate thousands of identifiers within the exact same millisecond tick, standard random generation can suffer from ordering ambiguity or out-of-sequence collisions.

Monotonic Generation resolves this by tracking generation state and incrementing the randomness component sequentially if subsequent generation requests occur within the same millisecond timestamp boundary.

Architectural Mechanics & State Synchronization

To preserve strict lexicographical sorting order across concurrent threads, the internal monotonic generator manages state synchronization and carry-over logic:

  1. Thread Synchronization: Utilizes a static synchronization lock (lock (SyncRoot)) to coordinate concurrent generation requests safely.
  2. Timestamp Observation:
  • If a new millisecond timestamp is observed (\(timestamp > \_lastTimestamp\)), the timestamp is updated and a fresh block of cryptographically secure randomness is seeded via RandomNumberGenerator.Fill(_random).
  • If multiple generation calls occur within the same millisecond (\(timestamp \le \_lastTimestamp\)), the timestamp is held/incremented and the internal 10-byte randomness component is sequentially incremented.
  1. Randomness Increment Strategy: The 80-bit random block is treated as an 80-bit big-endian integer, incrementing from the least significant byte (i = 9 down to 0) and automatically propagating carry values to higher-order bytes when required.

Zero-Allocation Performance

Like the standard generator, the monotonic engine constructs identifiers efficiently:

// Stack-allocated memory layout
Span<byte> bytes = stackalloc byte[16];
  • No Heap Pressure: Utilizing stack allocations (stackalloc) combined with fixed-index bit mapping ensures zero intermediate garbage collection overhead during tight generation loops.
