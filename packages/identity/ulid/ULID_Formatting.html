ULID Formatting (Ulid.Format)

📚 Navigation: Home > Dixor.Identity > Formatting

Dixor.Identity provides an allocation-optimized mechanism for converting raw 16-byte binary representations into canonical 26-character Universally Unique Lexicographically Sortable Identifiers (ULIDs) via the Ulid.Format(ReadOnlySpan<byte> bytes) facade.

using Dixor.Identity.ULID;

// Parse a ULID string into binary, then format it back into its canonical string representation
byte[] binaryId = Ulid.Parse("01J5J8Y9Q6A8K5J8X3Z5N9Y2KP");
string canonical = Ulid.Format(binaryId);

Fixed-Index Bit Mapping Strategy

To convert 128 bits of binary data into 26 Crockford Base32 characters (representing 130 total bits with 2 zero-padded trailing bits), the formatting engine uses a deterministic Fixed-Index Bit Mapping strategy rather than dynamic bit-shift loops:

128-bit Binary (16 bytes)
    ↓
Fixed-Index Bit Mapping
    ↓
26 Crockford Base32 Characters
  • Exact Alignment: This layout guarantees precise bit-alignment across the entire 128-bit structure, preventing offset drift and ensuring strict chronological sorting compatibility with external specification-compliant libraries.
  • Character Breakdown: Maps bytes directly into 5-bit chunks. The very first character contains 3 significant bits, with the remaining 2 bits serving as zero-padded padding.

Zero-Allocation Performance

To eliminate memory overhead during serialization, the formatter relies entirely on stack allocation:

// Allocates the 26-character result buffer directly on the call stack
Span<char> result = stackalloc char[26];
  • No Garbage Collection Pressure: Using stackalloc char[26] ensures that converting binary arrays into canonical strings generates zero heap allocations, maximizing performance in high-throughput data serialization pipelines.
