NanoId Generation (NanoId.Generate)

📚 Navigation: Home > Dixor.Identity > NanoId

Dixor.Identity provides compact, URL-friendly, and cryptographically secure identifiers generated via the Nano ID algorithm through the NanoId.Generate() facade.

using Dixor.Identity;

// Generate a secure NanoId of default length
string identifier = NanoId.Generate();

Algorithm & Execution Mechanics

The underlying NanoIdGenerator adapts the reference Nano ID algorithm for high-performance .NET execution, utilizing advanced bitmasking and rejection sampling:

  1. Optimized Bitmask Calculation: Evaluates the alphabet length using bit operations (BitOperations.Log2) to derive the smallest operational bitmask (mask), minimizing random byte waste.
  2. Rejection Sampling: Generates secure random bytes using RandomProvider.Fill(bytes) and applies bitwise masking (bytes[i] & mask). If the resulting index falls within the bounds of the alphabet (index < alphabet.Length), it selects the character; otherwise, it rejects the sample to ensure a strictly uniform distribution.
  3. Allocation Efficiency: Employs stack allocation (stackalloc) for character buffers and byte arrays up to 256 and 512 elements respectively, completely bypassing heap allocation overhead during generation.
// Example custom length invocation with parameter validation
string customId = NanoId.Generate(length: 21);

Guardrails & Validation Rules

To protect system integrity and prevent resource exhaustion, the underlying services enforce strict bounds:

  • Length Constraints: Passing a length less than or equal to zero, or greater than Constants.MaxIdentifierLength, triggers an ArgumentOutOfRangeException.
  • Alphabet Constraints: Alphabets must not be empty and cannot exceed 255 characters, throwing an ArgumentException otherwise.
