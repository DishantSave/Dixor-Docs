Sequence Generation (Sequence.Create)

📚 Navigation: Home > Dixor.Identity > Sequence

Dixor.Identity provides thread-safe, monotonically increasing integer sequences with configurable starting values and increments via the Sequence.Create() factory facade, suitable for generating application-specific identifiers like invoice numbers, order numbers, and ticket codes.

using Dixor.Identity;
using Dixor.Identity.Interfaces;

// Create a thread-safe sequence starting at 1 with an increment of 1
ISequenceService sequence = Sequence.Create(start: 1, increment: 1);

// Advance and retrieve sequential values
int nextVal = sequence.Next();
int currentVal = sequence.Current;

What are Sequential Generators & Why Use Them?

Sequential Integer Generators provide ordered numerical values that increment predictably over time.

Key Advantages

  • Strict Monotonic Order: Unlike random identifiers, sequences guarantee that every generated value is strictly greater than the last, which is essential for business objects that require chronological auditing (e.g., invoices, sequential order logs).
  • Concurrency-Safe: Built with thread-safe synchronization primitives, instances can be shared safely across multi-threaded web servers and background workers without race conditions or duplicate values.
  • Configurable Baselines: Allows custom starting offsets and step increments, enabling integration with legacy numbering systems or partitioned enterprise ID ranges.

Common Use Cases

  • Invoice & Order Tracking Numbers: Human-readable sequential reference numbers required for accounting, billing, and regulatory compliance.
  • Support Ticket & Case IDs: Predictable incremental identifiers that support easy verbal communication and customer service routing.
  • Batch Processing Counters: Local counters or pagination trackers used within asynchronous data pipelines.

Algorithm & Execution Mechanics

The underlying SequenceService and SequenceFactory manage independent, concurrent state using thread-safe synchronization:

  1. Atomic Sequence Advance: The Next() method utilizes low-level interlocked operations (Interlocked.Add) to ensure generated values remain strictly unique and sequential under heavy concurrent access from multiple threads and asynchronous tasks.
  2. Volatile State Reading: The Current property reads state via volatile memory reads (Volatile.Read) to safely retrieve the most recently generated value without lock contention.
  3. State Initialization: Each sequence instance maintains its own isolated counter state initialized via the factory.
// Example custom invocation with start and increment configuration
ISequenceService customSeq = Sequence.Create(start: 1000, increment: 5);

Guardrails & Validation Rules

To protect system integrity and ensure valid sequence behavior, SequenceFactory enforces strict bounds:

  • Starting Value Constraints: Passing a start value less than zero triggers an ArgumentOutOfRangeException.
  • Increment Constraints: Passing an increment value less than or equal to zero triggers an ArgumentOutOfRangeException.
