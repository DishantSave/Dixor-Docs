Changelog

All notable changes to the Dixor Ecosystem are documented in this file.

The format is based on Keep a Changelog, and this project adheres strictly to Semantic Versioning.

[Unreleased]

Planned

  • Dixor.Validation: DNS/MX record verification, disposable email detection, and RFC 3986 URL validation.

2026-07-04

Added — Dixor.Validation [v0.1.0] (Initial Release)

  • Email Validation Engine: Initial release of the state-machine email validation engine built without regular expressions to eliminate ReDoS vulnerabilities and reduce heap allocations.
  • Structured Diagnostics: Strongly typed error reporting (EmailValidationError) covering local-part syntax, domain labeling, consecutive dots, and illegal characters.
  • Suggestion Engine: Algorithmic domain correction engine that detects common mobile registration typos (e.g., suggesting gmail.com for inputs like gmai.com).
  • Parsing & Inspection: Dedicated APIs (EmailParser.Parse()) to safely extract and inspect structural email components (LocalPart, Domain).

Changed — Dixor.Identity [v0.1.4]

  • Architecture: Decoupled embedded Roslyn compile-time analyzers from the main Dixor.Identity runtime package. Analyzers will be distributed as an independent package to eliminate compiler tooling dependencies from the core runtime library.
  • Packaging: Reduced overall NuGet payload size by excluding analyzer binaries and updated package dependency metadata.

2026-06-28 — Dixor.Identity [v0.1.3]

Fixed

  • Packaging: Resolved package installation failures by embedding Roslyn analyzers directly into the package structure rather than referencing an external analyzer dependency.
  • Build: Improved analyzer integration and package reliability across Visual Studio, Rider, and CLI builds.

2026-06-28 — Dixor.Identity [v0.1.2]

Added

  • Multi-Targeting: Extended framework support beyond .NET 10 to fully support .NET 8.0 and .NET 9.0.
  • Documentation: Introduced modular DocFX navigation, cross-referenced conceptual guides, and complete API references for the Uuid7 facade.

Fixed

  • Compatibility: Resolved runtime assembly resolution issues when consuming the library from .NET 8/9 applications.
  • Documentation: Fixed broken navigation hierarchies and NuGet package Readme link rendering.

2026-06-28 — Dixor.Identity [v0.1.1]

Added

  • UUIDv7 Engine: Implemented RFC 9562 compliant UUID Version 7 generation (Uuid7.New()).
  • Monotonic Generation: Added lock-free sequential generation (Uuid7.NewMonotonic()) for high-throughput sub-millisecond workloads.
  • Batch APIs: Added high-efficiency lazy batch generation (Uuid7.NewGuids(count)).
  • Inspection & Formatting: Added timestamp extraction (Uuid7.GetTimestamp()), parsing, validation, and domain prefix/suffix formatting (Uuid7.NewString()).
  • Roslyn Analyzers: Introduced diagnostic rule DIXOR001 to catch non-positive batch size parameters during compilation.

2026-06-03 — Dixor.Identity [v0.1.0]

Added

  • Project Foundation: Created initial repository structure, core Dixor.Identity project architecture, automated unit test suites, and initial repository metadata.
